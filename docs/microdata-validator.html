Валидатор микроразметки

Инструмент помогает проверить, корректно ли поисковые роботы читают структурированные данные на странице.

Валидатор микроразметки позволяет быстро увидеть, как поисковые системы интерпретируют встроенную в страницу семантическую разметку. В один клик он проверяет страницу на наличие ошибок и предупреждений, подсказывает, какие элементы не распознаны, и помогает адаптировать данные под требования сервисов.

Сейчас валидатор поддерживает все популярные форматы разметки: микроданные (microdata), schema.org, микроформаты, OpenGraph и RDFa. Однако специальные сниппеты и использование размеченных данных в сервисах доступно не для всех форматов — актуальный перечень есть в разделах Справки Яндекса.

Как работать с валидатором

  1. Перейдите в валидатор микроразметки и укажите URL или вставьте фрагмент HTML-кода.
  2. Нажмите «Проверить» — сервис распарсит страницу и покажет найденные сущности.
  3. Изучите предупреждения: валидатор подскажет, какие обязательные поля отсутствуют и где разметку следует дополнить.

Подробнее о возможностях инструмента и поддерживаемых сценариях читайте в справке Яндекса.

Примеры сценариев

Адрес организации

Покажите поиску физический адрес компании, чтобы попасть в расширенный сниппет с картой.

Что важно указать: название организации, тип, координаты или адрес, контактный телефон и часы работы.

TVSeries

Схема помогает описать сериал: укажите сезон, эпизоды и участников съёмочной группы. Разметка улучшает карточку проекта и позволяет появиться в рекомендациях.

Результат проверки

После сканирования страница отчёта показывает, какие сущности нашёл валидатор и какие свойства им присвоены. Фрагмент отчёта для репозитория GitHub https://github.com/averinaleks/lending/tree/main/docs выглядит так:

Twitter Cards

  • twitter:image → изображение предпросмотра
  • twitter:site → @github
  • twitter:card → summary_large_image
  • twitter:title → «lending/docs at main · averinaleks/lending»
  • twitter:description → «Contribute to averinaleks/lending development by creating an account on GitHub.»

Open Graph

  • og:image → ссылка на изображение предпросмотра (1200 × 600 px)
  • og:image:alt → «Contribute to averinaleks/lending development by creating an account on GitHub.»
  • og:site_name → GitHub
  • og:title → «lending/docs at main · averinaleks/lending»
  • og:url → https://github.com/averinaleks/lending/tree/main/docs
  • og:description → «Contribute to averinaleks/lending development by creating an account on GitHub.»

Schema.org

  • @type → SoftwareSourceCode
  • author → averinaleks
  • name → lending

Дополнительные атрибуты

Сервис также фиксирует вспомогательные атрибуты ролей ARIA и ссылки на файлы opensearch.xml и favicon, которые помогают поисковым роботам распознать навигацию и элементы интерфейса.

Так парсер микроразметки обработает вашу страницу и наглядно покажет, какие блоки данных будут использованы поисковыми системами.

Материал подготовлен для владельцев сайтов, внедряющих структурированные данные.
