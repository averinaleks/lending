# Почему сайт okoroleva.com не открывается

## Возможные причины

1. **Домен не делегирован на DNS-серверы Reg.ru.** Для работы домена нужно указать корректные NS-записи. Если okoroleva.com не привязан к ns1.reg.ru и ns2.reg.ru, браузер не сможет получить IP-адрес и вернёт ошибку. Проверьте делегирование в панели Reg.ru и при необходимости исправьте настройки.
2. **Изменения DNS ещё не распространились.** После смены NS или ресурсных записей проходит до 24 часов, пока кэши DNS-серверов обновятся. Если записи настроены недавно, дождитесь окончания пропагации. Состояние можно отслеживать через сервисы DNSchecker или whatsmydns.
3. **Ошибки в A- или CNAME-записях.** Для корневого домена нужны A-записи с IP GitHub Pages: 185.199.108.153, 185.199.109.153, 185.199.110.153 и 185.199.111.153. Имя записи — @. Для поддомена www создайте CNAME на okoroleva.github.io. Убедитесь, что нет опечаток или конфликтующих записей. При желании добавьте AAAA-записи для IPv6: 2606:50c0:8000::153, 2606:50c0:8001::153, 2606:50c0:8002::153 и 2606:50c0:8003::153.
4. **Некорректная настройка DNSSEC.** Если для домена включён DNSSEC, устаревшие DS-записи могут приводить к ошибкам SERVFAIL. Обновите или отключите DNSSEC при смене NS.
5. **Локальный DNS-кэш или особенности провайдера.** Устаревшие данные резолвера могут мешать открытию сайта. Очистите локальный DNS-кэш (например, `ipconfig /flushdns` в Windows) или временно используйте публичные DNS 8.8.8.8/1.1.1.1.

## Как устранить проблему

1. Проверьте делегирование на ns1.reg.ru и ns2.reg.ru через WHOIS или панель управления доменом.
2. Убедитесь, что ресурсные записи соответствуют требованиям GitHub Pages и сохранены без ошибок.
3. Дождитесь полной пропагации DNS (обычно до суток), периодически проверяя записи через независимые сервисы.
4. После вступления изменений в силу очистите кэши браузера и DNS на устройстве и попробуйте открыть сайт снова.
5. Если спустя 24 часа проблема сохраняется, обратитесь в поддержку Reg.ru для дополнительной диагностики.
