Мониторинг поисковых запросов в Яндекс.Вебмастере

Подборка ключевых рекомендаций из справки Яндекса: как читать данные отчёта, настраивать фильтры и отличать падение спроса от проблем с видимостью.

Что показывает отчёт

Обзор
  • В отчёте собирается до 150 000 поисковых запросов, по которым сайт уже отображается в выдаче Яндекса.
  • Данные сгруппированы в два представления: «Запрос» показывает частотный URL для каждого запроса, «URL» — самый популярный запрос для выбранной страницы.
  • При переключении между представлениями Яндекс автоматически связывает запросы и страницы, поэтому удобно искать не только запросы, но и посадочные страницы.

Метрики и трактовка

Показатели
  • Показы фиксируют фактические появления сниппета на экране пользователя (включая динамические места). Потенциальные показы на следующих страницах не считаются.
  • Клики учитывают переходы со сниппета в органической выдаче и в динамических местах.
  • CTR рассчитывается по суммарным показам и кликам, помогает судить о привлекательности сниппета.
  • Позиция — среднее место выдачи, на котором виден сайт. Значение может отсутствовать, если сайт не показался, спрос равен нулю или пользователи не дошли до сайта.
  • Спрос отражает, насколько часто пользователи вводят запрос. Он доступен только в представлении «Запрос» и может быть выше суммарных показов, если сайт появляется несколько раз на первой странице.

Когда для одного запроса ранжируется несколько страниц, Яндекс суммирует показы и клики, позицию усредняет, а CTR считает по общей сумме.

Как работать с фильтрами

Сегментация
  • Используйте фильтры «включает», «не включает», «точное соответствие» или регулярные выражения, чтобы выбрать или исключить запросы и URL.
  • Фильтр «содержится в группе» подтягивает ранее сохранённые группы запросов из списка — удобно для повторных проверок.
  • Для метрик (показы, клики, позиция, CTR, спрос) задавайте условия «больше/меньше» на выбранном интервале — в таблице останутся строки, которые соответствовали условию во все дни.
  • Интервал мониторинга поддерживает последние две недели с разбивкой по дням или 18 месяцев с усреднением по месяцам.

При работе в представлении «URL» спрос не отображается — он доступен только по запросам, и фильтр по спросу тоже работает в этом режиме.

Проверка позиций и спроса

Практические шаги
  • Чтобы отслеживать видимость, выберите показатель «Позиция», при необходимости задайте регион и тип устройства — так проще понять, где проседает выдача.
  • Сравнивайте «Спрос» и «Показы», чтобы отделить сезонные колебания спроса от технических проблем. Для удобства выгружайте данные в XLS.
  • Фиксируйте сегменты запросов или URL и смотрите динамику спроса — помогает планировать контент и приоритизировать страницы.

Полезные заметки

Нюансы
  • Если позиция отсутствует, сначала проверьте значение спроса: ноль означает, что запрос не вводили, ненулевой спрос — сигнал проверить, нашёл ли пользователь сайт.
  • Подсказка в графике показывает точные значения по наведению — удобно отслеживать изменения в конкретную дату.
  • Для детальной аналитики по URL используйте переключатель представлений: в режиме «Запрос» легко найти связку запрос → страница, а в режиме «URL» — отследить, какие поисковые запросы тянут страницу вверх.
